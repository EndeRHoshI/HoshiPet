# HoshiPet - AI Development Context

> **To the next AI Assistant:** 
> Please read this document immediately upon loading the project. It contains the essential rules, mechanics, and current progress of the HoshiPet codebase to ensure seamless continuous development.

## 1. 架构与基本约定 (Architecture & Conventions)
- **技术栈**：纯静态前端 (HTML5 + CSS3 + Vanilla JS)。无构建工具，使用 Live Server 在本地 `5500` 端口调试测试响应式。
- **存储机制**：一切状态通过 JS 对象 `gs` 维护，由于没有后端服务器，每当关键操作发生（如喂食、学习、随时间流逝）都会调用 `saveGame()` 并序列化至浏览器的 `localStorage` 下。
- **UI 风格**：温暖治愈的日系毛玻璃 UI（全局 `.scene.active .card` 应用 `backdrop-filter: blur(8px)` 等），依赖 Emoji 作为各种模块的拟物化图标，不使用大体积的外部图标库。
- **语言**：项目逻辑、变量定义尽量使用清晰的英文单词驼峰命名，但界面展示文案、所有 Git Commit 信息以及当前此核心架构文档，均必须使用 **纯中文**。

## 2. 核心数学模型与游戏循环 (Core Math & Game Loops)
- **底层 Tick 系统**：游戏的基础时间刻度和驱动引擎，通过 `tickMs` (默认30000ms，即30秒) 触发一跳。UI 层面上，为了对玩家友好，游戏中体现的所有倒计时（如进度条显示“需 X 小时”）都是内部根据引擎直接套用公式 `tick * tickMs` 实时换算出**真实世界的物理时间**，并且绑定了系统里的“时间速度调整滑块”。
- **多日生存长期挂机框架**：
  - 猫咪的五维属性（饱食、水分、清洁、健康、心情）的自然流逝极其缓慢：完全饱食的情况下放置 **1 天（24小时）** 各属性大约只下降三十点左右，极限放置存活时间大约为 3 天。
  - **智能离线追帧 (Offline Logic)**：无需在后台死跑定时器。通过挂载浏览器的 `visibilitychange` 事件监听器，加上记录离开时的时间戳 `gs.lastSaveTime`，当玩家切回这个网页瞬间，会自动折算出中间流失的离线时长，并瞬间补全状态和日志（离线折算硬上限为挂机 8 小时，绝不过度惩罚）。
  - **深度时停机制**：调试面板中的全局变量 `gs.isPaused` 永久锁定时间状态并已与 `localStorage` 持久化绑定。开启时停哪怕直接关掉网页/杀进程，几个月后重开也完全恢复到定格那一刻的数值，真正实现了跨会话的“零损耗静止”。

## 3. 疲劳博弈与复杂经济系统 (Fatigue & Economy Balance)
- **精力损耗分级衰减（Fatigue Math）**：
  - **空闲在家 (Idle)**：如果吃得足够饱，健康和心情不仅不会下降，还会有**非常微弱的反向回血效果**，模拟养宠的舒适期。
  - **学习培训 (Study)**：中度疲劳行为。期间不再回血，并且健康和心情会发生被动式恶化衰减。
  - **打工赚钱 (Work)**：重度疲劳行为。每现实时间 2 个小时（240 Ticks）发放一次大额结算工资，但作为代价，期间的健康和心情状态会暴跌。若让猫咪打工一整夜，第二天回来必定处于重病/抑郁状态，借此强行逼迫玩家将宠物切换回“空闲”并配合高额医疗消费回温。
- **商城负面词条网课机制（Trade-offs）**：
  - 商城系统（20余种物品搭配折叠分类交互）完全脱离了“无脑买性价比高的食物续航”的传统点击游戏雷区。它现在是一个带有强烈生存策略倾向的“超市”。
  - 【经典案例】：喂食廉价的“纯牛奶”虽解渴却扣健康（乳糖不耐受）；使用“烤鸡腿”增加巨量饱食与心情，但同时弄脏身体加扣健康（肥胖）；给它洗“温水泡澡”必定解决肮脏状态并提升抵抗力，但因为怕水必定导致它心情产生负向跌落。因此，玩家必须通过物品组合来对抗五维跌落，不存在“万金油无负面”的平价物品（除开天价 SPA）。
- **场景化文案系统**：根据使用商品时猫咪当时的各项生命体征红线（例如：极度口渴 `thirst < 40` 时喂下饮品，文案会从常规的“解渴又舒爽”动态替换成“咕噜咕噜狂饮！”），外加精准的浮动气泡动画、底栏动作日志双记录反馈（日志后方准确附带如 `(水分+60 饱食-5 心情+40)` 等增减记录）。

## 4. 当前开发进度与下一步计划 (Current Status & Next Steps)
- **当前状态 (v1.0.x 稳定阶段)**：核心引擎模块、UI 响应式布局自适应处理、时间推演方程与离线追溯算法均已高度自闭环。数值经济模型的试运转目前处于逻辑高度自洽。
- **接下来立即接手的待办：全景出游系统重制**：
  - UI 顶层卡片和底部 Tab 栏拥有一个显眼的预留坑位：**外出（`sceneTrip`）**。
  - 这个面板目前是一张白纸状态的空壳界面。未来的主攻方向就是它！这里需要设计一整套类似《旅行青蛙》一样的出游逻辑、倒计时挂机、并辅以大量有趣的随机事件概率卡池（可能带回稀有的商品掉落、大量金钱或者因为打架被野猫抓伤导致生病等）。
