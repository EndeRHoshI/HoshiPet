# HoshiPet (星宠) 🐱✨

HoshiPet 是一款基于纯静态前端技术栈打造的**网页端宠物养成与放置生存游戏**。
游戏主打温暖治愈的“日系毛玻璃卡片”画风，拥有严密的物理时间换算系统、疲劳干预机制与长线物价经济学。你可以在电脑和手机浏览器上沉浸式体验饲养、打工、学习与收集的乐趣。

## 🛠️ 技术栈 (Tech Stack)
- **核心语言**: HTML5 / CSS3 / Vanilla Javascript (纯原生无框架，零构建依赖)。
- **UI 设计**: Flexbox / CSS Grid 响应式组合布局，配合前沿的 CSS 毛玻璃滤镜 (`backdrop-filter`) 与拟物化的系统原生 Emoji 作为图床。
- **数据持久化**: 深度依赖浏览器的 `Local Storage API` 搭配安全序列化机制，实现断电离线瞬间不丢档与跨会话读取。
- **环境要求**: 任何主流现代智能浏览器。

## 💡 核心实现原理 (Implementations)
1. **多级时间流速与 Tick Engine**:
   底层拥有一套精确的心跳算法（Tick Engine），将内部程序的离散 Tick 经过公式直接折算成对终端玩家友好的真实“时、分、秒”。配合 HTML5 的 `Page Visibility API`（页面可见性接口），在用户息屏、切到后台或睡眠时，系统会在页面再度可见的毫秒间执行**智能离线追帧算法**，一口气结算流失的离线时长与收益。
2. **多日生存模型与疲劳博弈 (Fatigue Economics)**:
   颠覆传统死板“无脑放置”。在完全待机时，生理指征（饱食、水分等）能安稳续航三日；而在执行“打工赚钱”或“跨级学习”时，基于事件状态机介入的疲劳压力算法，将使得猫咪的“健康”与“心情”发生剧烈消耗。
3. **动态制衡的商城策略体系 (Dynamic Trade-offs)**:
   打破了“平价补血”的固有套路。高热量零食大量补充饱腹感却必然折损健康且弄脏身体；给不想沾水的猫咪强行洗澡能提升大额抗性但必定导致心情闪崩；便宜的未过滤自来水暗藏致病隐患。此数学模型迫使玩家动态观测五维短板，组合配比不同档位的“零食、饮品、医疗与玩乐”服务打一套“连招组合拳”以维持长线存活。
4. **底层硬件级物理“时停”**:
   通过调试界面的全局单例开关控制整个事件总线的时间戳推进，实现了真正的硬盘级长线休眠。
