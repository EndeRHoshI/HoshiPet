# HoshiPet 开发与重构复盘日志 (Dev Log)

这份文档是我们这段时间结对编程开发 HoshiPet 的完整历史复盘。你可以将它保留在项目中随时查阅。

## 📅 版本迭代与开发历程

### 阶段一：底层机制从 0 到 1 的重构
最开始我们接手的是一个基础的单页面数值控制台，进行了脱胎换骨的改造：
- **数据结构升级**：废弃了简单的“吃喝玩乐”三维，升级为**五维状态系统**（饱食、水分、清洁、健康、心情），并引入了宠物天生个性的隐藏成长率（活泼度、抵抗力、智力）来动态影响五维自然衰减速度。
- **UI 框架化**：将干瘪的竖排页面解构为**多场景单页面应用 (SPA)**，通过 `switchScene` 逻辑顺滑切换 `小屋 (Home)`、`商城 (Shop)`、`异次元 (System)` 三个功能区。
- **离线收益与存储**：打通了浏览器的 `localStorage` 构建稳定存档。即使关闭网页，再次打开时通过 `applyOfflineTicks` 函数会计算离线流逝的时间，让宠物在你不在时也会感受到时间的流逝。

### 阶段二：美术与视觉表现力的飞跃
- **告别纯色**：通过 AI 生成工具为小屋和商城分别定制了“温馨卧室”和“日系货架”背景图。通过给 `body` 标签跨层级应用 CSS 背景，实现了完美贴合各种浏览器比例的**全屏自适应背景**。
- **生动的猫咪**：废弃了静态纯文字 Emoji，替换为 **4 张不同状态的定制插图**（打工、空闲、学习、出游）。
- **CSS 动效注入**：手写了多套 CSS 关键帧动画（`@keyframes`）。现在小猫在空闲时会“呼吸漂浮”，打工时会“努力摇晃”，学习时会“点头做笔记”，出游时会“欢快步行”。静止的数值游戏变成了活生生的动态养成。

### 阶段三：玩法深度拓展
- **职业技能系统（取代旧版技能点）**：
  构建了具有“天赋树”概念的全新职业系统：
  1. 初级工种（看门保安、咖啡师、捕鼠猎手）：只要挂机学习即可掌握。
  2. 进阶工种（例如侦探需要懂咖啡，星厨需要懂咖啡+捕鼠）：需要前置组合条件，有强烈的养成深度。
  3. 通过分离 `学习 (Study)` 和 `打工 (Work)` 状态，打工时可以选择已解锁的最高薪资职业上岗盈利（最高单次 38 金币），未入门只能打黑工（3 金币）。
- **商城精细化**：商品阵容一口气扩充到 20 种！拆分成了食品、饮料、清洁、医疗、娱乐、特殊（改名/复活卡）六大品类。
- **操作减负**：针对高频购买事件，优化了商品面板——由原先的弹窗确认改为了极其爽快的**一键点击立刻购买并扣款**。

### 阶段四：极限 Debug 与韧性优化
在后期全面融合这两大套系统时，我们遇到了几次因为**历史遗留档位脏数据**引发的致命空指针错误：
- 💀 `updateProfessionUI` **渲染假死**：旧档中未初始化 `learnedProfessions`，导致进入职场页面时地图遍历函数奔溃。
- 💀 `offlineNotice` **白屏死机**：为了做全图全屏背景，误删了 `index.html` 里的离线流逝外壳，导致底层代码找不到画板而罢工。
- 💀 `gameAvatar` **渲染阻断**：在升级猫咪插图时，原文字猫咪 DOM 节点被移除，但在主循环更新 `updateUI` 结算函数时仍在强行给不存在的兵长赋值，导致刚领养的点击事件瞬间熔断。

**🛡️ 我们的防线（解决方案）：**
通过引入内嵌 Chrome 无头浏览器抓取最底层的 JS 堆栈报错日志，我们采取了最彻底的恢复手段：
我们在入口函数 `DOMContentLoaded` 里埋入了**强力异常捕获熔断机制 `try...catch`**。从此以后，不论是因为更新代码结构、或者是玩家乱改了本地代码，只要检测到存档反序列化失败或是逻辑跑不通，系统将静默启动“清道夫”，**自动销毁损坏的旧档并安全退回到 `Adopt` 新生主界面**，彻底治愈了“无限白屏卡死”的疑难杂症。
